概念

PEM：Privacy Enhanced Mail，即增强隐私邮件，是一种主要用于SSL/TLS通信、数字签名、加密通信等领域的文件格式。

DER： 是一种广泛用于各种加密和安全通信协议中的二进制编码格式，尤其是X.509证书和PKCS#8私钥的存储和传输中。

关系： PEM是DER证书的base64编码格式

类型

根据传输的内容不同，PEM能被分为多种文件，主要包括以下三种

1.证书

2.公钥

3.私钥

数据拆分：
私钥的格式如下：

  • 3082 025c： 30变送SEQUENCE类型，025c指下面文件内容的字节数（0x025c是572字节）
  • 02 01 00： 版本，02表示整型，01表示长度为1字节，00表示版本号为0
  • 02 81 81： 模数n，02表示整型，长度为129字节（0x81）
  • 02 03： 公钥指数e，02表示整型（此位置若是03表示 BIT STRING），03长度为3字节
  • 02 81 80： 私钥指数d，0x80表示长度128字节
  • 02 41： 素数p，0x41表示长度65字节
  • 02 41： 素数q，同上
  • 02 40： $d \mod p-1$，长度0x40即64字节
  • 02 40： $d \mod q-1$，长度同上
  • 02 41： $q^{-1}\mod p$，长度同上

ps：注意在处理不完整的base64时，要通过删除一些数据来保证正确解码
